## 이용 전에 알아두세요

### 심사 방식

- AI가 **업로드한 글**만 읽습니다. 앱을 **실행하지는 않습니다.**
- 결과는 **참고용**이며, 공식 최종 심사와 다를 수 있습니다.
- **세 분야는 독립 평가**됩니다. 한 분야만 잘 써도 다른 분야 점수는 그대로입니다.

### 걸리는 시간

- 보통 **30초~2분**. 문서가 길면 **3분**까지 갈 수 있습니다.
- 「심사 시작」은 **한 번만** 누르고 기다리세요.

## 1. 429, Rate limit (일시적 한도 초과)

### 증상

화면에 아래와 비슷한 메시지가 보입니다.

```
OpenAI API 오류: Error code: 429
Rate limit reached for gpt-4o ... tokens per min (TPM)
Please try again in ...s
```

또는 「잠시 후 다시 시도」 안내.

### 왜 생기나요?

채점은 내부적으로 **AI(대형 언어 모델)** 에게 기획서·README·코드 전체를 읽히고 평가를 요청합니다. 이때 처리하는 **글자 양(토큰)** 이 한 번에 많습니다.

AI 서비스는 **짧은 시간에 처리할 수 있는 양에 상한**이 있습니다. 비유하면:

> 식당 주방에 **1분에 만들 수 있는 음식 수**가 정해져 있는데,  
> 짧은 시간에 **주문이 몰리거나**, **한 주문(문서)이 너무 크면**  
> 「잠시만 기다려 주세요」라고 하는 것과 같습니다.

**흔한 원인**

| 원인 | 설명 |
|------|------|
| **연속 클릭** | 「심사 시작」을 여러 번 눌러 **동시에 여러 심사**가 돌아감 |
| **문서가 매우 김** | 세 파일 합쳐 읽을 양이 많아 **한 번에 한도**에 걸림 |
| **행사 시간대에 이용 집중** | 여러 팀이 **비슷한 시간**에 채점하면 서버·AI 쪽 부하 증가 |
| **방금 다른 팀이 채점함** | 직전 1분 안에 이미 많은 분량이 처리된 상태 |

**중요:** 여러분 파일이 「틀렸다」는 뜻이 **아닙니다.** **일시적으로 처리 줄이 길어진 것**입니다.

### 해결 방법

1. **1~2분 기다린 뒤** 「심사 시작」을 **딱 한 번**만 누르기
2. 이미 눌렀다면 **추가 클릭 금지** — 진행 중일 수 있음
3. 기획서·README·코드가 **매우 길다면** 핵심만 남긴 버전으로 재시도
4. 행사 피크 시간이면 **5~10분 뒤** 다시 시도
5. 반복되면 **운영 스태프**에게 알리기 (동시 이용 팀 수·시각 함께 전달)

### 다른 사람에게 설명할 때

> 「우리 파일이 잘못된 게 아니라, AI 심사 서버가 **1분에 처리할 수 있는 양을 잠깐 초과**한 거예요.  
> **1~2분 기다렸다가 버튼을 한 번만** 다시 누르면 됩니다.  
> 문서가 너무 길면 줄이는 것도 도움이 됩니다.」

### 예방

- 「심사 시작」 **한 번만** 클릭
- 채점 전 **제출_파일_양식.md** 체크리스트로 파일 정리
- 가능하면 팀원이 **동시에 여러 번** 누르지 않기

## 2. 파일 업로드 실패

### 증상

- 파일을 골랐는데 **칸에 파일명이 안 뜸**
- 「**요청 형식이 올바르지 않습니다**」 (422 오류)
- 「심사 시작」 후 바로 실패

### 왜 생기나요?

채점 페이지는 브라우저에서 파일을 **텍스트(글자)** 로 읽어 서버에 보냅니다. 아래 경우에는 읽기·전송이 실패합니다.

| 원인 | 설명 |
|------|------|
| **파일 3개 중 하나 누락** | 기획서·README·코드 중 빈 칸이 있음 |
| **지원하지 않는 확장자** | `.docx`, `.pdf`, `.hwp`, `.zip` 등은 **자동 읽기 불가** |
| **빈 파일** | 내용이 없거나 거의 없음 |
| **페이지 상태 꼬임** | 업로드 중 새로고침·뒤로가기 등으로 브라우저 상태가 어긋남 |

**왜 Word/PDF는 안 되나요?**  
Word·PDF는 겉보기와 달리 **내부가 복잡한 형식**입니다. 이 채점 도구는 **메모장에 적은 것 같은 순수 텍스트**만 처리하도록 만들어져 있습니다.

### 해결 방법

1. **세 칸 모두** 파일이 선택됐는지 확인 (파일명이 각 칸에 표시되는지)
2. 확장자 확인:

   | 칸 | 허용 |
   |----|------|
   | 기획서 | `.md`, `.txt`, `.markdown` |
   | README | `.md`, `.txt`, `.markdown` |
   | 실행 코드 | `.py` |

3. Word로 썼다면 → **다른 이름으로 저장 → UTF-8 텍스트(.txt)** 또는 Markdown(.md)
4. **F5 새로고침** 후 파일 **다시 선택**
5. 그래도 안 되면 **다른 브라우저**(Chrome, Edge)로 시도

### 다른 사람에게 설명할 때

> 「채점 사이트는 **글자 파일만** 받을 수 있어요.  
> 한글/워드 문서는 **메모장이나 VS Code로 .md / .txt 로 저장**해서 올려야 합니다.  
> 기획서·README·app.py **세 개 다** 넣었는지도 확인해 주세요.」

### 예방

- 처음부터 `.md` 또는 `.txt`로 작성
- 업로드 전 **제출_파일_양식.md** 체크리스트 확인

## 3. 심사 시간 초과

### 증상

- 「심사 시작」 후 **2~3분 이상** 기다려도 결과 없음
- `The operation was aborted`, 네트워크 오류, 시간 초과 메시지

### 왜 생기나요?

| 원인 | 설명 |
|------|------|
| **서버 절전 해제 중** | 오랫동안 아무도 안 쓰다가 **첫 요청**이면 깨어나는 데 시간 소요 |
| **문서가 너무 김** | AI 처리 + 전송 시간 증가 |
| **네트워크 불안정** | 행사장 Wi-Fi 혼잡, VPN 등 |
| **중복 요청** | 여러 번 눌러 서버가 동시에 여러 작업 처리 |

브라우저는 보통 **약 3분**까지만 응답을 기다립니다. 그 안에 AI 심사가 끝나지 않으면 **실패로 보일 수** 있습니다.

### 해결 방법

1. **1분 기다린 뒤** 페이지 새로고침 → 파일 다시 올리기 → **한 번만** 심사 시작
2. Wi-Fi·유선 연결 확인, 가능하면 **다른 네트워크** 시도
3. 문서·코드 **분량 줄이기** (불필요한 로그·주석·긴 붙여넣기 제거)
4. 절전 해제가 원인이면 **첫 시도는 느리고, 두 번째는 빨라질 수** 있음

### 다른 사람에게 설명할 때

> 「첫 채점은 서버가 잠에서 깨어나서 **1분 정도 느릴 수** 있어요.  
> 그리고 문서가 너무 길면 **3분 안에 끝나지 않아** 끊길 수 있습니다.  
> **조금 기다렸다가 한 번만** 다시 시도하고, 파일이 길면 **핵심만** 올려 보세요.」

## 4. 특정 분야 0점 또는 「심사 불가」

### 증상

- 종합 점수는 나오는데 **한 분야만 0점**
- 「해당 분야 심사 불가」 류 안내

### 왜 생기나요?

심사 전에 시스템이 **「이 글이 기획서/README/코드로 쓸 만한가」** 를 먼저 검사합니다. 아래면 그 분야만 심사를 건너뜁니다.

| 분야 | 0점·불가가 나는 흔한 이유 |
|------|---------------------------|
| **공공기관 적합성** | 기획서가 너무 짧음, 목적·요구사항이 없음, 기획서가 아닌 내용 |
| **의도 구현도** | 코드가 너무 짧음, Python 코드가 아님, 기획서와 **완전히 다른 주제** |
| **README 품질** | README가 너무 짧음, 설치·실행 안내가 없음, README가 아닌 내용 |

**예시 (흔한 실수)**

- 기획서 칸에 `안녕하세요`, `test`만 있는 파일
- README 칸에 기획서와 **다른 과제** README
- 코드 칸에 **메모만** 있고 `import`, `def` 같은 코드가 없음

### 해결 방법

| 분야 | 보완 방법 |
|------|-----------|
| 공공기관 적합성 | 목적, 페인포인트, 기능 표, 성공 기준, 범위 외 추가 |
| 의도 구현도 | 기획서와 **같은 프로젝트**의 `app.py`인지 확인, 기능 구현 여부 점검 |
| README 품질 | `pip install …`, `streamlit run app.py`, 프로젝트 구조 추가 |

수정 후 **다시 업로드**하면 됩니다.

### 다른 사람에게 설명할 때

> 「0점은 **망한 게 아니라**, 그 칸 파일이 **심사할 수 있는 형태가 아니었다**는 뜻이에요.  
> 예를 들어 README에 **실행 방법이 없거나** 너무 짧으면 README 분야만 심사가 안 됩니다.  
> **제출_파일_양식.md** 보고 그 분야만 고치면 됩니다.」

## 5. 음성 재생 실패

### 증상

- 점수·글 코멘트는 **정상 표시**
- 음성 버튼만 실패, 「음성 생성에 실패」 등

### 왜 생기나요?

음성은 **채점과 별도**로, 결과 글을 소리로 바꾸는 **추가 기능**입니다. 채점이 끝난 뒤 **또 한 번** 서버·브라우저를 사용합니다. 이 단계만 실패해도 **글 결과는 이미 저장·표시된 상태**입니다.

| 원인 | 설명 |
|------|------|
| 음성 서버 일시 오류 | TTS(글→소리) 변환만 실패 |
| 브라우저 음성 제한 | 자동 재생 정책, 스피커·음소거 |
| 네트워크 순간 끊김 | 음성 파일 다운로드 실패 |

### 해결 방법

1. 화면에 **「브라우저 음성으로 재생」** 안내가 있으면 그대로 사용
2. 페이지 새로고침 → 결과 다시 띄운 뒤 음성 버튼 재시도
3. **글 결과만** 확인해도 채점 목적은 달성됨

### 다른 사람에게 설명할 때

> 「채점은 **이미 끝났고**, 점수랑 코멘트는 다 나왔어요.  
> **소리로 읽어 주는 기능만** 실패한 거라, 글만 봐도 됩니다.」

## 6. 서비스 설정 오류

### 증상

「OpenAI API 키가 설정되지 않았습니다」 등 **서비스·설정** 관련 메시지

### 왜 생기나요?

참가자 파일 문제가 아니라, **채점 서버 쪽 AI 연결 설정**이 비어 있거나 잘못된 경우입니다. 행사 준비 과정에서 서버 환경을 맞춰야 하는 **운영 측 이슈**에 가깝습니다.

### 해결 방법 (참가자)

1. **잠시 후** 다시 시도 (일시적일 수 있음)
2. **계속되면** 운영 스태프에게 **오류 화면 캡처** + **발생 시각** 전달
3. 다른 팀도 같은지 확인 → 여러 팀 동시면 **서버 쪽 문제** 가능성 큼

### 다른 사람에게 설명할 때

> 「우리 파일 문제가 아니라 **채점 서비스 서버 설정** 문제로 보입니다.  
> 운영팀에 **스크린샷이랑 시간** 알려 주면 됩니다.」

## 7. 첫 접속이 느릴 때

### 증상

- 사이트 첫 로딩이 **30초~1분** 걸림
- 첫 「심사 시작」만 유독 느림

### 왜 생기나요?

웹 서비스가 **한동안 사용이 없으면 절전**에 들어갑니다. 누군가 접속하면 **다시 켜지는 시간**이 필요합니다. 무료 호스팅에서 흔하며, **고장이 아닙니다.**

### 해결 방법

1. **30초~1분** 기다리기
2. 새로고침 후 다시 시도
3. 한 번 깨어난 뒤에는 **같은 세션에서는 빨라지는** 경우가 많음

### 다른 사람에게 설명할 때

> 「사이트가 잠깐 **잠들어 있다가 깨어나는 중**이라 첫 접속만 느린 거예요.  
> **1분만 기다리면** 정상 속도로 돌아옵니다.」

## 8. 점수가 낮게 나왔을 때

### 왜 생기나요?

오류가 아니라, AI가 제출 내용을 기준에 맞춰 **객관적으로 낮게 평가**한 경우입니다.

| 흔한 이유 | 어떤 분야에 영향 |
|-----------|------------------|
| 기획서·코드·README **서로 다른 이야기** | 의도 구현도, README |
| 기획 **성공 기준**이 코드에 없음 | 의도 구현도 |
| README에 **실행 방법** 없음 | README 품질 |
| 기획서에 **현장 문제** 설명 부족 | 공공기관 적합성 |
| 기획에 **없는 기능**을 코드에 추가 | 간결성, 의도 구현도 |

### 해결 방법 (재채점)

1. 결과의 **리스크** 항목 읽기 — AI가 지적한 부분이 힌트
2. **제출_파일_양식.md** 체크리스트로 수정
3. 수정한 파일로 **다시 업로드** 후 심사 (429 주의: **한 번만** 클릭)

### 다른 사람에게 설명할 때

> 「에러가 아니라 **피드백**이에요.  
> 예를 들어 README에 실행 방법이 부족하면 README 점수만 낮게 나옵니다.

## 9. 그래도 안 될 때 (에스컬레이션)

운영 스태프에게 알릴 때 **아래 네 가지**를 함께 전달하면 해결이 빠릅니다.

| # | 전달 내용 |
|---|-----------|
| 1 | **오류 메시지 전체** 캡처 (또는 그대로 복사) |
| 2 | **언제** 발생했는지 (시각) |
| 3 | 올린 **파일 이름** 3개 + **대략 분량**(짧음/보통/김) |
| 4 | **이미 시도한 것** (1분 대기, 새로고침, 파일 형식 변경 등) |

**재시도 규칙:** 같은 조건으로 **2~3분 간격**을 두고 **한 번만** 시도한 뒤, 반복되면 위 정보와 함께 문의하세요.
