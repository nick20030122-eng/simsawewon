# 인공지능 심사위원 AI JUDGE — 사용 방법

> **이 안내는 참가팀·이용자를 위한 문서입니다.**  
> 별도 설치나 API 키 없이, 웹 브라우저만 있으면 이용할 수 있습니다.

---

## 1. 이 서비스는 무엇을 하나요?

**인공지능 심사위원 AI JUDGE**는 여러분이 준비한 **기획서**, **README**, **실행 코드(`app.py`)**를 AI가 읽고, 아래 3가지 관점에서 점수와 코멘트를 알려 주는 **사전 점검·참고용** 도구입니다.

| 평가 분야 | 무엇을 보나요? |
|-----------|----------------|
| **공공기관 적합성** | 기획서 — 현장 문제, 해결 방향, 적용 가능성 |
| **의도 구현도** | 기획서와 실행 코드가 서로 맞는지 |
| **README 품질** | README — 설치·실행 방법, 문서 완성도 |

- 각 항목은 **0~100점**입니다.
- Pass/Fail(합격·불합격)은 표시하지 않습니다.
- **최종 심사 결과를 대신하지 않습니다.** 제출 전 스스로 점검하는 용도로 활용해 주세요.

### 왜 세 가지 파일을 모두 올려야 하나요?

이 서비스는 **한 파일만** 보는 것이 아니라, 세 자료를 **서로 대조**합니다.

| 비유 | 설명 |
|------|------|
| 기획서 | 「무엇을 만들기로 했는지」 적어 둔 **설계도** |
| 실행 코드 | 「실제로 무엇을 만들었는지」 보여 주는 **완성품** |
| README | 「다른 사람이 어떻게 써야 하는지」 적어 둔 **설명서** |

설계도와 완성품이 다르거나, 설명서와 실제 사용법이 다르면 감점 요인이 됩니다. 그래서 **세 파일이 모두 필요**하고, **같은 프로젝트**를 가리켜야 합니다.

## 2. 어디서 이용하나요?

웹 브라우저(Chrome, Edge 등)로 아래 주소에 접속합니다.

| 페이지 | 주소 | 하는 일 |
|--------|------|---------|
| **홈** | https://innocurve-judge.onrender.com | 시작 화면 |
| **채점** | https://innocurve-judge.onrender.com/evaluate.html | 파일 업로드·심사 |
| **채점 기준** | https://innocurve-judge.onrender.com/criteria.html | 평가 기준 미리 보기 |

상단 메뉴 **홈 · 채점 · 채점 기준**으로도 이동할 수 있습니다.

## 3. 심사가 진행되는 방식 (내부 흐름)

「심사 시작」을 누르면 대략 아래 순서로 처리됩니다. 알아 두면 **왜 시간이 걸리는지**, **왜 버튼을 여러 번 누르면 안 되는지** 이해하기 쉽습니다.

```
[여러분 PC]  세 파일 선택
      ↓
[여러분 PC]  파일 내용을 글자(텍스트)로 읽음
      ↓
[인터넷]     채점 서버로 전송
      ↓
[채점 서버]  AI에게 기획서·README·코드를 읽히고 평가 요청 (3회 분석 + 총평)
      ↓
[채점 서버]  점수·코멘트를 정리해 화면에 표시
      ↓
[여러분 PC]  결과 확인 (+ 선택: 음성 재생)
```

**알아 두면 좋은 점**

- AI가 여러분 컴퓨터에서 앱을 **실행하지는 않습니다.** 글만 읽습니다.
- 기획서·README·코드를 **각각 따로** 평가한 뒤, 서로 맞는지도 봅니다.
- 문서가 길수록 AI가 읽을 양이 많아져 **시간이 길어지고**, 가끔 **일시적 오류**(429 등)가 날 수 있습니다. → 자세한 내용은 `주의사항_및_문제해결.md`

## 4. 이용 순서 (처음 오신 분)

### ① 채점 기준 먼저 읽기 (권장)

**채점 기준** 페이지에서 다음을 확인해 보세요.

- 4대 핵심 가치(정확성·독창성·완전성·간결성)가 어떻게 반영되는지
- 기획서·README·코드 각각에서 무엇을 보는지

**왜 먼저 읽나요?**  
점수가 「왜 이렇게 나왔는지」 이해하려면, AI가 **무엇을 기준으로 보는지**를 미리 아는 것이 도움이 됩니다.

### ② 파일 준비하기

| 순서 | 파일 | 형식 |
|------|------|------|
| 1 | 기획서 | `.md` 또는 `.txt` |
| 2 | README | `.md` 또는 `.txt` |
| 3 | 실행 코드 | `.py` (보통 `app.py`) |

자세한 작성 방법은 **제출_파일_양식.md**를 참고하세요.

### ③ 홈에서 채점으로 이동

1. 홈 화면의 **망치(법봉) 로고**를 클릭하거나, 메뉴에서 **채점**을 누릅니다.

### ④ 파일 업로드

- **클릭**하거나 **드래그 앤 드롭**으로 선택
- 세 칸이 **모두** 채워져야 합니다

**왜 확장자가 제한되나요?**  
채점 시스템은 **글자로 된 텍스트 파일**만 읽을 수 있게 만들어져 있습니다. Word·PDF·압축 파일은 내용을 자동으로 읽지 못합니다.

### ⑤ 심사 시작

1. **「심사 시작」** 버튼을 **한 번** 누릅니다.
2. 보통 **30초~2분** 정도 기다립니다.
3. **기다리는 동안 버튼을 연속으로 누르지 마세요.**

**왜 연속 클릭이 문제인가요?**  
한 번 누를 때마다 서버에 **새 심사 요청**이 들어갑니다. 여러 번 누르면 같은 작업이 **동시에 여러 번** 돌아가서, 대기 시간이 늘거나 **429(일시적 한도 초과)** 오류가 날 수 있습니다.

### ⑥ 결과 확인

- **종합 점수**와 3대 분야 점수
- 세부 항목 점수표
- **잘한 점** / **리스크** / **최종 한마디**

**점수 해석 팁**

| 상황 | 의미 |
|------|------|
| 한 분야만 낮음 | 그 분야 파일(기획서·README·코드)을 보완하면 됨 |
| 의도 구현도만 낮음 | 기획서와 `app.py` 내용이 어긋났을 가능성 |
| README만 낮음 | 실행 방법·구조 설명이 부족했을 가능성 |
| 세 분야 모두 낮음 | 파일이 너무 짧거나, 서로 다른 프로젝트일 가능성 |

### ⑦ 음성으로 듣기 (선택)

결과 화면의 음성 버튼으로 심사 코멘트를 들을 수 있습니다.  
음성은 **부가 기능**이므로, 실패해도 **글 결과는 그대로** 유효합니다.

---

## 5. 더 정확한 점검을 위한 팁

1. **기획서·README·`app.py`는 반드시 같은 프로젝트**여야 합니다.
2. README에 적은 실행 방법대로 **직접 한 번 실행**해 본 뒤 업로드하세요.
3. 기획서에 **없는 기능**을 코드에만 넣으면 「의도 구현도」·「간결성」에 불리합니다.
4. **세 분야는 따로** 채점됩니다. README만 잘 써도 기획서 점수는 올라가지 않습니다.

## 6. 자주 묻는 질문

**Q. AI가 제 앱을 직접 실행해 보나요?**  
A. 아니요. **업로드한 글(문서·코드)** 만 읽습니다. 그래서 README에 실행 방법을 잘 적어 두는 것이 중요합니다.

**Q. 같은 파일을 여러 번 올리면 점수가 같나요?**  
A. 비슷하지만, AI 특성상 **조금씩 달라질 수** 있습니다. 한 번 결과를 보고 파일을 고친 뒤 **다시 올리는 것**이 목적에 맞습니다.

**Q. 처음 접속이 느려요.**  
A. 한동안 사용이 없으면 서버가 **절전 상태**였다가 깨어납니다. **30초~1분** 기다린 뒤 다시 시도해 보세요. (무료 웹 호스팅에서 흔한 현상입니다.)

**Q. 팀원에게 오늘 겪은 문제를 설명하려면?**  
A. **주의사항_및_문제해결.md**에 증상·원인·해결·「다른 사람에게 설명할 때」 예시 문장이 정리되어 있습니다.
