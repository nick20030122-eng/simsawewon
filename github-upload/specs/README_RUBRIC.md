# README 평가 규칙 (심사 분야 3)

README만으로도 프로젝트 품질·완성도·운영 준비 상태를 판단할 수 있어야 합니다.
아래 규칙에 따라 0~100점으로 채점합니다.

---

## 세부 항목

### 3-1. 설치·실행 안내 (setup_instructions)

| 점수대 | 기준 |
|--------|------|
| 90~100 | OS/환경, 의존성 설치, 환경변수, 실행 명령까지 누락 없이 재현 가능 |
| 70~89 | 실행 가능하나 일부 전제(버전, 경로)가 암묵적 |
| 50~69 | 개요만 있고 실제 실행 단계가 불충분 |
| 0~49 | 실행 방법 없음 또는 오류 가능성이 큰 안내 |

### 3-2. 기획·코드 정합성 (documentation_accuracy)

| 점수대 | 기준 |
|--------|------|
| 90~100 | README의 기능 설명·입출력·범위가 기획서·실행 코드와 일치 |
| 70~89 | 대체로 일치, 사소한 누락·표현 차이 |
| 50~69 | README와 기획/코드 간 불일치가 눈에 띔 |
| 0~49 | README가 실제 프로젝트와 동떨어짐 |

### 3-3. 유지보수·확장 가이드 (maintainability)

| 점수대 | 기준 |
|--------|------|
| 90~100 | 프로젝트 구조, 핵심 파일 역할, 확장 시 주의점이 명확 |
| 70~89 | 구조 설명은 있으나 확장·운영 가이드가 얕음 |
| 50~69 | 파일 목록 수준, 역할 설명 부족 |
| 0~49 | 구조·유지보수 정보 없음 |

---

## README에 있으면 가산 요소

- 환경변수(`.env`) 설명
- 프로젝트 디렉터리 구조
- 범위 외(미구현) 명시
- 알려진 제한사항

## README만으로 판단이 어려운 경우 (감점 사유)

- 실행 파일(`app.py`)이 분리 모듈에 의존하는데 README에 구조 설명 없음
- 기획서 기능과 README 설명 불일치
- 설치 명령만 있고 실제 진입점·포트 정보 없음

---

## 분야 3 종합 점수

`(setup_instructions + documentation_accuracy + maintainability) / 3`
